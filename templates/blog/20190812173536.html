FTPサーバvsftpdの導入

(CentOS5.7で確認) 　FTPとは、ファイル転送のためのプロトコルです。FTPクライアントとしては、WindowsではFFFTPが有名です。Linux上で動くFTPサーバとしてvsftpdの導入方法を解説します。 ##vsftpdのパッケージ ``` vsftpd ``` ##インストール 　以下のコマンドでインストールできます。 ``` yum -y install vsftpd ``` ##インストールの確認 　rpmコマンドでパッケージがインストールされたかを確認する。 ``` rpm -q vsftpd ``` ##サービス起動スクリプト 　サービス起動スクリプトを見ればいろいろな情報がわかります。 ``` # サービス起動スクリプトを見る。 view /etc/init.d/vsftpd ``` ##サービスの自動起動の設定 　自動起動の設定を確認する。 ``` # 自動起動の確認 chkconfig --list vsftpd # 自動起動をonにする。 chkconfig vsftpd on # サービスそのものとして登録されていない場合は追加する。 chkconfig --add vsftpd ``` ##サービスの開始・停止・確認 ``` # サービスが開始しているかどうかの確認 service vsftpd status # 開始していない場合は開始 service vsftpd start # 停止する場合は service vsftpd stop ``` ##接続確認 　おそらくvsftpを立ち上げた時点で接続できると思われます。まずpingでサーバ自体が生きているかを確認。 ``` ping サーバのIPアドレス ``` Windowsだとftpコマンドがデフォルトついているのでftp接続してログインしてみる。 ``` ftp サーバのIPアドレス ``` 　ログイン名を聞かれるので、anonymous で匿名ログイン。パスワードは空。すでにLinuxでのログインユーザを作成しているならその名前でログインできます。 ##設定ファイル ###vsftpdの設定ファイル ``` /etc/vsftpd/vsftpd.conf ``` ##ログイン禁止ユーザ名 　デフォルトでは、rootなどのユーザではFTPログインできません。ログイン禁止ユーザは、以下のファイルに書かれています。ftpusersというファイル名ですが、FTPログインを禁止するユーザ名のリストなので注意します。 ``` /etc/vsftpd/ftpusers ``` ##ファイル転送に関するログ 　ファイル転送が行われた場合のログは、デフォルトの設定であれば、 ``` /var/log/xferlog ``` に出力されます。 ##アクセス制御設定ファイルの編集 　特定のIPアドレスだけからのアクセスを許可するようにしたい場合は以下のようにします。 ###/etc/hosts.deny 　まずvsftpdへのアクセスをすべて拒否する。 ``` vsftpd: ALL ``` ###/etc/hosts.allow 　次に必要なものだけアクセスを許可する(必要に応じて設定)。 ``` # (例)192.168.1.0 ～ 192.168.1.255 までを許可 vsftpd: 192.168.1. ```
